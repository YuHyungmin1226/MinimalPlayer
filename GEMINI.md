# 저사양 최적화 포터블 플레이어 프로젝트 지침

## 1. 개요
본 프로젝트는 저사양 PC에서도 원활하게 동작하는 동영상/오디오 플레이어를 목표로 합니다. `mpv` 엔진을 기반으로 하며, `PySide6`를 통해 테두리 없는 미니멀 다크 UI를 제공합니다. 별도의 설치 과정 없이 실행 파일만으로 동작하는 **포터블 설계**를 핵심 원칙으로 삼습니다.

설치·실행·빌드 절차의 상세 명령어는 `README.md`를 참고하십시오. 이 문서는 "왜 이렇게 설계했는가"와 "무엇이 구현되어 있는가"를 다룹니다.

## 2. 시스템 아키텍처 (mpv 라이브러리 로딩)
mpv(libmpv) 로딩 방식은 OS별로 다르게 설계되어 있으며, 이는 각 플랫폼의 배포 관행 차이를 반영합니다.

- **Windows**: 포터블 배포를 위해 `mpv-1.dll`을 저장소에 포함하지 않고, `main.py` 최초 실행 시 GitHub Release에서 자동 다운로드하여 실행 파일과 같은 폴더에 배치합니다.
  - 다운로드된 DLL은 **SHA256 해시 검증**을 거친 후에만 사용됩니다(`mpv_setup.py: verify_mpv_dll`). 해시가 일치하지 않으면 파일을 삭제하고 재다운로드를 유도하여, 손상되거나 변조된 바이너리를 로드하지 않도록 합니다.
  - 빌드(PyInstaller) 시 프로젝트 루트에 `mpv-1.dll`이 이미 존재하면 실행 파일 내부에 자동 번들링되어, 최종 사용자는 다운로드 팝업 없이 완전한 오프라인 실행 파일을 사용할 수 있습니다.
- **macOS**: DLL 자동 다운로드 방식 대신, Homebrew로 설치된 시스템 `libmpv`(`libmpv.dylib` 등)를 탐색하여 사용합니다(`find_libmpv_dir`). `brew --prefix`, `/opt/homebrew/lib`, `/usr/local/lib` 순으로 탐색 후 `DYLD_FALLBACK_LIBRARY_PATH`에 등록합니다. 빌드 시에는 PyInstaller가 libmpv와 그 의존 라이브러리(ffmpeg, libass 등)를 `.app/Contents/Frameworks`에 재귀적으로 번들링해 Homebrew 없는 Mac에서도 실행 가능한 자기완결형 앱을 만듭니다.
- **Linux**: 시스템에 설치된 `libmpv.so` 계열을 표준 라이브러리 경로(`/usr/lib`, `/usr/local/lib` 등)에서 탐색하여 `LD_LIBRARY_PATH`에 등록합니다. 별도 자동 다운로드는 제공하지 않으며, 패키지 매니저(`apt`, `dnf` 등) 설치를 전제로 합니다.
- 라이브러리 로드 실패 시 플랫폼별로 적절한 안내 메시지(Homebrew 설치 안내, apt/dnf 안내, DLL 수동 배치 안내)를 다이얼로그로 표시합니다.

## 3. UI 구성
- **타이틀 바**: 기본 OS 윈도우 프레임을 제거(`FramelessWindowHint`)하고, 자체 구현한 타이틀 바(앱 이름, 최소화 `-`, 닫기 `x` 버튼)를 사용합니다. 타이틀 바나 영상 영역을 드래그하여 창을 이동할 수 있습니다.
- **비디오/오디오 영역**: `QStackedWidget`으로 두 화면을 전환합니다.
  - 비디오 트랙이 있으면 OpenGL 기반 `MpvGLWidget`(mpv render context 직접 렌더링)을 표시합니다.
  - 비디오 트랙이 없는(오디오 전용) 파일이면 자동으로 오디오 뷰로 전환하여, 커버 이미지(있으면) 또는 `♪` 아이콘을 표시하고, 감지된 자막이 있으면 화면 하단에 오버레이 자막을 렌더링합니다.
- **컨트롤 바**: 하단 고정 바에 재생바(클릭 시 즉시 해당 지점으로 이동하는 Jump-to-Click 슬라이더), Open/뒤로10초/재생·일시정지/앞으로10초 버튼, 시간 표시, 볼륨 라벨 및 슬라이더를 배치합니다.
- 전체적으로 반투명한 어두운 톤(`#121212`, `#1e1e1e` 등)과 얇은 구분선만 사용하여 콘텐츠에 집중하는 미니멀리즘을 유지합니다.
- 창 크기 조절 시 영상 비율을 유지하며 컨트롤 요소가 유연하게 재배치됩니다.

## 4. UX 및 단축키
| 기능 | 조작 |
| --- | --- |
| 재생/일시정지 | `Space` |
| 5초 탐색 | `←` / `→` |
| 10초 탐색 | 컨트롤 바 `<<` / `>>` 버튼 |
| 볼륨 조절 | `↑` / `↓` (OSD 표시) 또는 볼륨 슬라이더 |
| 자막 싱크 조정 | `Z`(+0.1초) / `X`(-0.1초) (OSD 표시) |
| 자막 크기 조절 | `[`(축소) / `]`(확대) (OSD 표시) |
| 전체화면 토글 | `Enter` 또는 영상 더블클릭 |
| 종료 | `ESC`(창 모드) 또는 `x` 버튼 |
| 파일 열기 | Open 버튼, 드래그 앤 드롭, 또는 최근 파일 메뉴 |

- **드래그 앤 드롭**: 지원되는 미디어 파일을 창에 끌어다 놓으면 즉시 로드됩니다.
- **전체화면 자동 숨김**: 전체화면 모드에서 마우스가 3초간 움직이지 않으면 타이틀 바·컨트롤 바·커서가 자동으로 숨겨지며, 컨트롤 영역 위에 있으면 유지됩니다.
- **우클릭 컨텍스트 메뉴**: 파일 열기, (오디오 재생 중이면) 동영상으로 내보내기, 최근 파일 목록/비우기, (Windows 전용) 기본 프로그램으로 설정, 종료를 제공합니다.
- **볼륨/화면 크기**는 `QSettings`에 앱 재시작 간 유지됩니다(볼륨은 종료 시 저장, 창 크기는 기본 1000x600 고정 시작).
- 재생 종료(EOF) 시 재생바를 끝까지 채우고 Play 버튼으로 되돌리며, 종료 상태에서 뒤로 이동하면 다시 재생 가능한 상태로 전환됩니다.

## 5. 자막 자동 인식
- 영상/오디오 파일과 **동일한 폴더**, **동일한 파일명**을 가진 자막 파일(`.srt`, `.ass`, `.vtt`, `.smi`, 대소문자 구분 없음)을 자동으로 감지하여 로드합니다. 여러 확장자가 동시에 존재하면 `SUBTITLE_EXTENSIONS` 순서(srt > ass > vtt > smi)로 우선순위를 적용합니다.
- **SMI 다국어 처리**: SMI 자막은 여러 언어 트랙이 한 파일에 섞여 있는 경우가 많아, `KRCC`/`KORCC` 클래스를 우선 추출하고 없으면 첫 번째로 발견된 언어 클래스를 사용해 SRT로 변환한 뒤 재생합니다(`utils.py: convert_smi_to_srt_text`).
- **인코딩 자동 복구**: UTF-8이 아닌 자막 파일(주로 CP949/EUC-KR로 저장된 한국어 자막)은 `utf-16` → `cp949` → `euc-kr` 순으로 디코딩을 시도하여 UTF-8 임시 파일로 변환 후 재생합니다. 변환된 임시 파일은 재생 종료/파일 전환 시 자동 삭제됩니다.

## 6. 최근 파일 및 이어보기
- 최근 재생한 파일은 `QSettings`에 최대 `RECENT_FILES_LIMIT`(5개)개까지 경로 목록으로 저장되며, 우클릭 메뉴의 "Recent Files"에서 다시 열거나 "Clear Recent Files"로 전체 삭제할 수 있습니다. 존재하지 않는 파일은 목록 정규화 시 자동으로 제외됩니다.
- 재생 위치는 파일별로 저장되며, 저장 시점 기준 `RESUME_THRESHOLD_SECONDS`(10초) 이상 재생했고 끝부분(마지막 5초 이전)이 아니면 위치가 기록됩니다. 파일 경로는 SHA256 해시로 정규화하여 설정 키를 만들므로, 경로에 포함된 구분자 문자로 인한 키 충돌이 없습니다.
- 같은 파일을 다시 열면 저장된 위치가 임계값 이상일 경우 "이어보기" 여부를 묻는 다이얼로그를 표시하고, 동의 시 해당 지점부터 재생을 시작합니다.

## 7. 오디오 → 비디오 내보내기 (ffmpeg 하드번)
- 오디오 파일 재생 중 우클릭 메뉴의 "Export to MP4 Video..."를 통해, 오디오 + 커버 이미지 + (있는 경우) 자막을 하나의 MP4 파일로 합성할 수 있습니다.
- 커버 이미지는 `find_matching_image`로 자동 탐색(동일 파일명 → cover/folder/album/front/artwork 순)하며, 찾지 못하면 사용자가 직접 이미지 파일을 선택하도록 안내합니다.
- 감지된 자막이 있으면 ffmpeg `subtitles` 필터로 **하드번(영상에 굽기)** 처리합니다. SMI 등 비-SRT 자막은 먼저 SRT/UTF-8로 변환한 뒤 사용합니다.
- 시스템에 `ffmpeg` 실행 파일이 PATH에 있어야 동작하며, 없을 경우 안내 다이얼로그를 표시하고 기능을 비활성화합니다.
- 인코딩은 `QProcess`로 백그라운드 실행하며, ffmpeg의 `-progress pipe:1` 출력을 파싱하여 진행률 다이얼로그에 반영합니다. 사용자는 언제든 취소할 수 있으며, 이미 내보내기가 진행 중일 때 다시 실행을 시도하면 완료를 기다리거나 취소하라는 안내와 함께 무시됩니다(동시 실행 방지).

## 8. 파일 연결 (Windows 전용)
- 우클릭 메뉴의 "Set as Default App" 또는 `python main.py --register` CLI 플래그로 Windows 레지스트리(`HKEY_CURRENT_USER\Software\Classes`)에 앱을 등록하고, 지원하는 모든 확장자(`MEDIA_EXTENSIONS`)에 대해 파일 연결 후보로 노출시킵니다.
- Windows 정책상 실제 "기본 프로그램" 지정은 사용자가 설정 앱에서 수동으로 완료해야 하므로, 등록 완료 후 `ms-settings:defaultapps` 페이지를 자동으로 열어줍니다.
- macOS/Linux에서는 파일 연결 기능을 제공하지 않으며, `--register` 실행 시 Windows 전용 기능임을 안내하고 종료합니다.
